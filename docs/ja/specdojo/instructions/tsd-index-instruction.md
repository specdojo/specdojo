# 技術スタック定義（TSD Index）作成の指示テンプレート

## 1. 目的と前提

- **生成対象**: 技術スタック定義インデックス (`tsd-index.md`)
- **目的**: プロダクトが採用する技術の全体インベントリ（一覧表）を一箇所に集約し、採用技術の担当責務を関係者間で合意すること。
- **対象外**: 特定技術の詳細設定、構成手順、コードブロック、および採用検討中の技術。これらは個別の `tsd-<term>.md` で定義する。

## 2. 入力情報

- **必要情報**:
  - 採用が確定している技術の一覧
  - 各技術のカテゴリ（区分）
  - 各技術のメジャーバージョン
  - 各技術をシステム内でどのような役割（担当責務）で使用するか
- **未確定時の扱い**: 採用が確定していない技術は含めない。検討中のものは `_UNDECIDED_:` としてメモを残すか、本ドキュメントからは除外する。

## 3. 出力フォーマット

### 3.1. Frontmatter

以下の項目を必須で含めること。

- `id`: `tsd-index` (固定)
- `type`: `architecture` (固定)
- `status`: `draft` / `ready` / `deprecated`
- `rulebook`: `tsd-index-rulebook`

### 3.2. 本文構成

以下の章構成で作成すること。

- `## 1. 技術スタック一覧`
  - 採用技術を網羅した単一の表を配置する。
- `## 2. 補足`
  - 詳細ドキュメント (`tsd-<term>.md`) へのリンクや、全体に関する注意事項を記載する。

## 4. 記述ルール

### 4.1. 技術スタック一覧表の構成

表は以下の列を必須で含めること。

| 列名     | 記述内容                                                                      |
| :------- | :---------------------------------------------------------------------------- |
| 区分     | カテゴリ（例: 言語 / ランタイム, DB / MQ / Cache, CI/CD / パッケージ配信 等） |
| 採用技術 | 技術名とメジャーバージョン（例: `TypeScript 5.x`, `Node.js 24.x`）            |
| 担当責務 | その技術を何に使うか、判定可能な粒度で1行で記述する                           |

- 必要に応じて `補足` 列を追加してよい。
- 区分ごとに表を分割せず、必ず1つの表にまとめること。

### 4.2. 補足章の記述

- 詳細な設定手順や構成詳細は記述せず、`tsd-<term>.md` へのリンクを記載することに留める。

## 5. 禁止事項

- **重複記載**: 同じ技術を複数の区分に重複して記載しない。
- **責務の空欄**: `担当責務` を空欄にしない。
- **機密情報の記載**: 接続情報、API キー、機密 URL、パスワード等を記載しない。
- **詳細の展開**: `tsd-<term>.md` に記載すべき詳細内容（設定手順、コードブロック等）を本文に展開しない。
- **未確定技術の記載**: 採用が確定していない技術（候補・検討中）を記載しない。

## 6. 最終チェック

- [ ] Frontmatter の `id` が `tsd-index` になっているか。
- [ ] `## 1. 技術スタック一覧` に単一の表が配置されているか。
- [ ] 表の列に `区分`, `採用技術`, `担当責務` がすべて含まれているか。
- [ ] `担当責務` に空欄がなく、具体的かつ簡潔に記述されているか。
- [ ] 採用確定済みの技術のみが記載されており、検討中のものは除外されているか。
- [ ] 機密情報や詳細な設定手順が混入していないか。
