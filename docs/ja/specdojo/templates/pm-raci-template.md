# RACI

本書は、プロジェクトにおける主要成果物と主要プロセスの責任分担を、PM が計画化、進捗確認、課題・リスク管理に使える粒度で定義する。

## 1. 目的

本 RACI は、成果物と主要プロセスごとに、作成責任、承認責任、相談先、共有先を分離し、Schedule、課題、リスク、変更要求、決定記録へ接続できる責務境界を示す。
PM は本 RACI を、タスク化、順序付け、所要時間見積もり、進捗確認、課題化、リスク登録、変更要求の影響整理に利用する。

採用する Role code の語彙は [[_PROJECT_ID_:pm-organization|組織定義]] を正とする。RACI は、Role code の一覧や実行主体の割り当てを複製する文書ではなく、採用済み Role code を使って責任分担を確認する文書である。

## 2. 適用方針

- RACI の列には、[[_PROJECT_ID_:pm-organization|組織定義]] で採用した Role code のみを使用する。
- 採用済み Role code のうち、本マトリクスに含めるロールを定義する（一部を省略する場合はその理由を記載する）。
- `A` (Accountable) は 1 成果物・1 プロセスにつき 1 Role code に限定する。
- Agent は `R` (Responsible) または `C` (Consulted) の支援を行ってよいが、`A` は担わない。
- 実際の member、agent、兼務割り当ては [[_PROJECT_ID_:pm-members|メンバー定義]] を正本とし、本書には複製しない。

## 3. RACI の定義

| 記号 | 意味        | 説明                                         |
| ---- | ----------- | -------------------------------------------- |
| R    | Responsible | 実作業を担当する                             |
| A    | Accountable | 最終責任を持ち、承認または判断する           |
| C    | Consulted   | 作成前または判断前に相談・レビューへ参加する |
| I    | Informed    | 結果、変更、決定の共有を受ける               |

## 4. 成果物別 RACI

成果物行は、PM が Schedule や進捗確認で管理する単位に合わせる。各行の `A` は最終判断または承認責任、`R` は作成・更新の実作業責任、`C` は確認者・合意形成の参加者、`I` は共有対象を示す。

| 成果物                     | _ROLE_1_ | _ROLE_2_ | _ROLE_3_ |
| -------------------------- | -------- | -------- | -------- |
| _TODO_: `<deliverable-id>` | A/R      | C        | I        |
| _TODO_: `<deliverable-id>` | C        | A/R      | C        |

## 5. プロセス別 RACI

プロセス行は、進捗、課題、リスク、変更要求、決定記録へ転記する管理・報告単位に合わせる。成果物作成、レビュー、変更判断、公開判断など、最終責任者が異なるものは行を分ける。

| -------------------------------------- | -------- | -------- | -------- |

## 6. 見直し条件

| 更新トリガー                                                      | 見直し内容                                                             |
| ----------------------------------------------------------------- | ---------------------------------------------------------------------- |
| [[_PROJECT_ID_:pm-organization\|組織定義]] で採用ロールを変更した | RACI 列、全行の `A`、Schedule の `owner` との整合を確認する            |
| 実行主体または兼務割り当てを変更した                              | [[_PROJECT_ID_:pm-members\|メンバー定義]] と RACI の責務境界を確認する |
| 成果物カタログまたは WBS の対象成果物を大きく変更した             | 成果物別 RACI の行を追加、削除、統合する                               |
| Schedule の `owner`、順序、所要時間見積もりを大きく変更した       | `R` / `A` と進捗確認・報告単位、課題・リスク登録への影響を確認する     |
| 進捗遅延、課題滞留、リスク顕在化が継続した                        | PM の `R` と `A`、エスカレーション先、報告対象を見直す                 |

## 7. 禁止事項

- [[_PROJECT_ID_:pm-organization|組織定義]] で採用していない Role code を RACI 列に使わない。
- member nickname、人名、agent 名を RACI 列に使わない。
- Agent に `A` を割り当てない。
- 各行の `A` を省略しない。
- 1 行に複数の `A` を置かない。
- 兼務を理由に、最終判断者である `A` を曖昧にしない。
- Schedule の `owner`、課題・リスク・変更要求の管理単位と矛盾する責任分担を記載しない。
