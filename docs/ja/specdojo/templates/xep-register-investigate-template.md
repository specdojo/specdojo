# Edit Plan: _PJR_ID_ _PJR_TITLE_

## 1. このタスクで行うこと

プロジェクト登録簿の項目 _PJR_ID_「_PJR_TITLE_」（_PJR_TYPE_）について調査し、結論案を result に記録する。成果物やソースコードの編集は行わない。

_PJR_DESCRIPTION_

## 2. 対象項目

- `id`: _PJR_ID_
- `type`: _PJR_TYPE_
- `priority`: _PJR_PRIORITY_
- `owner`: _PJR_OWNER_
- `due`: _PJR_DUE_
- `登録簿`: `_PJR_INDEX_PATH_`
- `個票`: _PJR_TICKET_REF_
- `result`: `_RESULT_REF_`

## 3. 進め方

1. 登録簿の該当行と個票（存在する場合）を読み、問い・リスク内容・判断に必要な条件を把握する。
2. 関連する成果物・設計書・実装・実行ログを確認し、判断材料を集める。
3. 実行 agent が Web 検索能力を持ち、一般的な観点・用語・標準の確認が必要な場合は、関連情報を取得して出典を添える。
4. 集めた材料から結論案を組み立てる。question は回答案、risk は影響・発生可能性の評価と対応案（回避・軽減・受容など）を示す。
5. 判断しきれない事項は、不足している情報と確認方法を result の申し送りに残す。

## 4. 完了手順

1. 「進め方」に従って調査を完了する。
2. result の実施内容に調査の要約と結論案を、変更ファイルに `-`（編集なし）を、申し送りに人間が判断・実施すべき事項を記入する。これはタスク完了に必須であり、_TODO_ を残したまま終了しない（詳細は共通規約を参照）。
3. 登録簿（`pjr-index.md`）の行と個票のステータス項目・結論列は変更しない。結論の確定と状態遷移は人間が行う。

## 5. 異常終了の条件

- 前提不足・判断材料の欠落により調査自体が成立しない場合は異常終了する（終了コード 1）。
- 標準エラー出力に理由を出力する（例: `blocked: <reason>; need=<next action>; ref=<path>`）。
- agent 自身は登録簿の状態を変更せず、終了コードと標準エラー出力で runner に結果を返す。
