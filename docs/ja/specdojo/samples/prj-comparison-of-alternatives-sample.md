# 代替案比較: 駄菓子屋きぬや 販売管理システム

本書は、店頭販売、在庫、つけ管理を継続しやすくするための実現方針を比較し、採択理由を残す最小の完成例である。

## 1. 比較目的と前提条件

比較目的は、単一店舗で無理なく導入でき、店主代表以外が店番しても販売・在庫・つけの記録を継続できる実現方針を選ぶことである。

根拠資料は次のとおり。

- `prj-scope-sample`: 店頭販売、在庫、つけ管理を初期対象にする境界。
- `prj-issues-and-approach-sample`: 最小機能のタブレット記録を中核にする解決方針。

前提条件は次のとおり。

- 初期リリースは単一店舗、1 台のタブレット利用を前提にする。
- 店主代表が最終的な業務判断を行い、システムは記録と確認を支援する。
- EC、本格会計、仕入れ先連携は初期対象外とする。
- 初期段階では、記録継続性と操作の簡単さを優先する。

## 2. 比較対象案

| 案ID   | 案                 | 概要                                                         | 対象範囲                         |
| ------ | ------------------ | ------------------------------------------------------------ | -------------------------------- |
| ALT-01 | タブレット記録優先 | 販売、在庫、つけを同じ端末で記録し、日次確認できるようにする | 販売登録、在庫確認、つけ残高確認 |
| ALT-02 | 紙様式の統一優先   | ノートと紙の様式を揃え、店番交代時の記録漏れを減らす         | 記録様式、運用手順、確認表       |
| ALT-03 | 在庫分析優先       | 先に在庫の集計と発注判断を整え、品切れと過剰在庫を減らす     | 在庫集計、在庫低下一覧           |

## 3. 評価軸と評価基準

| 評価軸     | 重み | 判定基準                                       | 備考             |
| ---------- | ---- | ---------------------------------------------- | ---------------- |
| 効果       | 5    | 記録漏れ、確認時間、欠品の改善に効くか         | 業務価値との対応 |
| コスト     | 3    | 小規模な開発・運用で始められるか               | 低いほど高評価   |
| 期間       | 3    | 初期リリースまでに成果が出るか                 | 短いほど高評価   |
| リスク     | 4    | 記録停止、誤操作、移行漏れのリスクが低いか     | 低いほど高評価   |
| 運用適合   | 4    | 店主代表と家族利用者代表が日々の店番で使えるか | 高いほど高評価   |
| 技術実現性 | 4    | 1 台のタブレットと最小データ管理で実現できるか | 高いほど高評価   |

評価は `High` / `Middle` / `Low` で行う。いずれも「その評価軸で採択判断に有利か」を示す。総合評価は機械的な加重合計ではなく、根拠資料との整合と業務要件への影響を踏まえて判断する。

評価軸は `prj-issues-and-approach-sample` の主要課題（記録漏れ・在庫過不足・つけ残高確認の負荷・継続運用の困難）に対応させる。効果は各課題への改善効果を見て、リスクと運用適合は店主代表や家族利用者代表による継続利用の観点で評価する。

## 4. 比較結果と採択理由

| 案ID   | 効果   | コスト | 期間   | リスク | 運用適合 | 技術実現性 | 総合評価 | 判定     |
| ------ | ------ | ------ | ------ | ------ | -------- | ---------- | -------- | -------- |
| ALT-01 | High   | Middle | Middle | Middle | High     | High       | High     | 採択     |
| ALT-02 | Middle | Low    | High   | Middle | Middle   | High       | Middle   | 一部採択 |
| ALT-03 | Middle | Middle | Low    | Middle | Middle   | Middle     | Middle   | 一部採択 |

採択案は ALT-01 とし、ALT-02 の紙様式は移行期の控えとして一部採択する。ALT-03 は販売・在庫記録が継続できることを確認した後に強化する。

採択理由:

- 販売、在庫、つけ残高を同じ手順で記録でき、店番交代時の記録漏れを減らせる。
- 1 台のタブレットで始められ、初期リリースの範囲に収まる。
- 紙様式を控えとして残すことで、操作に慣れるまでの記録停止リスクを下げられる。

技術的実現性と影響:

| 案ID   | 技術的実現性                             | 主な影響                                                                 |
| ------ | ---------------------------------------- | ------------------------------------------------------------------------ |
| ALT-01 | 最小データ項目と単一端末運用で実現できる | 販売・在庫・つけの記録形式が揃い、後続の受入条件とテスト観点を作りやすい |
| ALT-02 | 紙だけで始められる                       | 導入は容易だが、検索や集計は手作業のまま残る                             |
| ALT-03 | 記録データが蓄積すれば実現しやすい       | 先に進めると、入力負荷が増えて記録継続が不安定になる                     |

非採択または一部採択の理由:

| 案ID   | 扱い     | 理由                                             | 再評価条件                                    |
| ------ | -------- | ------------------------------------------------ | --------------------------------------------- |
| ALT-02 | 一部採択 | 紙だけではつけ残高確認と在庫集計の負荷が残る     | タブレット操作の定着に時間がかかる場合        |
| ALT-03 | 一部採択 | 記録データが少ない初期段階では効果を確認しにくい | 記録が2週間継続し、在庫低下の傾向が読める場合 |

## 5. リスクとトレードオフ

| 区分         | 内容                                                     | 軽減策                                                   |
| ------------ | -------------------------------------------------------- | -------------------------------------------------------- |
| トレードオフ | 初期は会計連携や仕入れ先連携を扱わず、記録継続を優先する | 次フェーズ候補として管理し、初期リリース後に再評価する   |
| リスク       | タブレット操作に慣れず、営業中の記録が止まる             | 紙様式を控えとして残し、初期運用期間に操作メモを確認する |
| リスク       | 既存ノートからのつけ残高移行に漏れが出る                 | 移行前後で店主代表が残高確認し、差分を記録する           |

案ごとのリスクとトレードオフ:

| 案ID   | 主なリスク / トレードオフ                                                  | 軽減策 / 扱い                                                |
| ------ | -------------------------------------------------------------------------- | ------------------------------------------------------------ |
| ALT-01 | タブレット操作に慣れる前に、営業中の記録が止まるリスクがある               | 紙様式を控えとして残し、操作に慣れるまでの切り戻し手段とする |
| ALT-02 | 紙様式のみでは集計・検索が手作業のまま残り、後続のデジタル化で再整理が必要 | タブレット定着後の移行計画を初期から意識しておく             |
| ALT-03 | 記録データが少ない段階では在庫傾向の読み取りが困難で、判断精度が低くなる   | 記録が2週間継続したことを確認してから強化する                |

## 6. 決定と見直し

- 現時点の採択方針は「タブレット記録優先、紙様式を移行期の控えとして併用」とする。
- 採択案の最終判断は `PO` が行う。AI Agent は比較表の作成支援に留める。
- 記録が2週間継続しない場合、紙様式中心への一時切り戻しを検討する。
- 方針変更時は、判断理由と影響範囲を決定記録または変更要求ログへ残す。
