# プロジェクト課題と解決アプローチ: 駄菓子屋きぬや 販売管理システム

本書は、単一店舗の店頭販売を継続しやすくするための主要課題と解決アプローチを示す最小の完成例である。店頭販売・在庫・つけ管理を初期対象とするスコープと、単一店舗・タブレット 1 台・店主の最終判断という前提を踏まえて記述する。

## 1. 課題一覧

| 課題ID | 課題                                                   | 影響                                               | 優先度 | 備考                               |
| ------ | ------------------------------------------------------ | -------------------------------------------------- | ------ | ---------------------------------- |
| P-01   | 店番が交代すると販売・在庫・つけの記録方法が揃わない   | 記録漏れと確認のやり直しが増え、残高が信用できない | High   | 店主代表以外でも同じ手順で使いたい |
| P-02   | つけ残高を手書きで管理しており、確認に時間がかかる     | 残高の誤りや回収漏れが発生する                     | High   | 常連顧客の残高確認を簡単にしたい   |
| P-03   | 在庫の減りに気づくのが遅く、品切れと過剰仕入れが起きる | 売上機会の損失と廃棄が増える                       | Medium | 在庫低下を日次で把握したい         |

## 2. 原因（仮説でも可）

| 課題ID | 原因区分 | 内容                                                   | 根拠 / 補足                  |
| ------ | -------- | ------------------------------------------------------ | ---------------------------- |
| P-01   | 事実     | 記録手順が口頭で共有され、文書化されていない           | 店番交代時の確認で判明       |
| P-02   | 事実     | つけ残高がノート管理で、集計と検索に手間がかかる       | 月末確認の所要時間から       |
| P-03   | 仮説     | 在庫確認のタイミングが決まっておらず属人的になっている | 品切れ発生時の状況からの推測 |

## 3. 解決策候補

| 案ID | 案                             | 概要                                                 | 期待できる効果                         |
| ---- | ------------------------------ | ---------------------------------------------------- | -------------------------------------- |
| A-01 | 最小機能のタブレット記録を導入 | 販売・在庫・つけを同じ手順で記録できる最小機能を作る | 店番が交代しても記録が揃う             |
| A-02 | 紙の運用を整えて様式を統一     | 記録様式と手順書を紙で統一し、システム化はしない     | 導入は容易だが集計と検索は手作業のまま |

## 4. 採用アプローチと理由

採用アプローチは、A-01 を中核とし、初期は記録継続に必要な最小機能に絞る方針とする。

| 採用対象              | 判定   | 理由                                                     |
| --------------------- | ------ | -------------------------------------------------------- |
| A-01 タブレット記録   | 採用   | 店番交代後も記録が揃い、残高と在庫の確認を継続できるため |
| A-02 紙運用の様式統一 | 非採用 | 集計と検索の負荷が残り、つけ残高の確認が早くならないため |

判断軸の優先順位は、記録継続性、操作の簡単さ、導入のしやすさの順とする。

## 5. トレードオフ / リスク

| 区分         | 内容                                               | 軽減策                                   |
| ------------ | -------------------------------------------------- | ---------------------------------------- |
| トレードオフ | 初期は最小機能に絞り、会計連携や分析は後回しにする | 次フェーズの候補として記録しておく       |
| リスク       | 店主代表がタブレット操作に慣れず記録が止まる       | 操作メモを用意し、初期運用期間に確認する |
