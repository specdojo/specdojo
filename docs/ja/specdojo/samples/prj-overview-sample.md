# プロジェクト概要: 駄菓子屋きぬや 販売管理システム

駄菓子屋きぬや 販売管理システムは、店頭販売、在庫、常連顧客のつけを一元管理し、店主一人でも日々の運用を続けやすくするための小規模店舗向け業務支援システムである。

## 1. 背景

地域の子どもたちに長年親しまれている「駄菓子屋きぬや」では、販売記録や在庫管理をノートと記憶に頼っている。店主の高齢化により、仕入れ判断や在庫確認、つけ残高の確認に時間がかかる場面が増えている。

現在、次の問題が生じている。

- 人気商品（うまい棒、ラムネ等）の欠品が増えている
- 仕入れのタイミングや数量が勘頼りで、在庫のムラが大きい
- 常連の子どもや近所の大人のつけ管理が曖昧で、記録漏れや確認コストが発生している
- 店主以外が店番をした際に、販売・在庫・つけの状況をすぐに確認できない

## 2. 必要性

このまま記憶と紙の運用を続けると、欠品や在庫過多による損失が増え、店舗継続リスクが高まる。また、店主以外が店番をする場合に同じ判断ができず、業務が属人化したままになる。

とりわけ繁忙期は来店と取引が集中し、欠品・在庫・つけの確認負荷が一気に高まる。記憶と紙の運用のままでは繁忙期の負荷に対応しきれず、損失と属人化のリスクが顕在化するため、負荷が高まる前のこの時期に着手する必要がある。

## 3. 基本的な考え方

1. **Record First（まず記録する）**: 高度な分析よりも、販売・在庫・つけの記録が毎日続くことを優先する。
2. **Simple Operation（迷わず操作できる）**: 店主一人でも使えるように、画面数と入力項目を最小限にする。
3. **Small Start, Improve Later（小さく始めて改善する）**: 初期リリースでは単一店舗の店頭運用に絞り、記録が安定してから仕入れ支援や分析を拡張する。

## 4. 実現したいこと

- 駄菓子の販売、在庫、常連顧客のつけをシンプルな仕組みで一元管理する。
- 店主一人でも無理なく操作できる画面と操作フローを提供する。
- 家族が店番をする場合でも、在庫状況とつけ残高を同じ手順で確認できるようにする。
- 欠品や過剰在庫に気づける販売・在庫の記録基盤を整え、後続の仕入れ支援や分析につなげられるようにする。

## 5. 期待効果

本プロジェクトの短期的な成功は、店頭販売、在庫確認、つけ残高確認の最小運用が成立し、店主が日々の記録を継続できる状態になることで判定する。売上改善や店舗継続への影響は、運用開始後に継続的に確認する中長期的な効果として扱う。

下記の目標値は初期仮説であり、実測値に基づき継続的に見直す。初期リリース時点では機能の達成度を判定し、日々の操作時間と運用が定着するまでの立ち上がり期間は、初期運用期間（リリース後2週間を想定）の実測値で判定する。

### 5.1. 短期指標

短期的には、初期リリースで最小運用を立ち上げ、店頭販売・在庫確認・つけ残高確認が現場で成立するかを、QCD（品質・コスト・納期）の観点で確認する。

| 観点     | 指標                                                           | 目標                  | 測定方法                                                     |
| -------- | -------------------------------------------------------------- | --------------------- | ------------------------------------------------------------ |
| Quality  | 初期リリースに必要な販売・在庫・つけ管理機能の達成度           | 90%以上               | 成功基準と受入条件で定めた項目に対して、実現できた割合を確認 |
| Cost     | 1営業日の販売・在庫・つけの記録・確認にかかる店主の操作時間    | 1日あたり合計10分以内 | 初期運用期間に、記録・確認の所要時間を日次で計測             |
| Delivery | 初期リリースから最小運用が毎営業日定着するまでの立ち上がり期間 | 2週間以内             | 初期運用期間に、販売・在庫・つけの記録有無を日次で確認       |

### 5.2. 中長期指標

中長期的には、運用開始後の店舗運営への効果を確認する。運用直後は母数が小さいため、まず実績件数を記録し、一定期間の記録が集まった後に削減率などの成長指標へ移行する。

| 観点       | 指標                         | 初期目標        | 測定方法                                 |
| ---------- | ---------------------------- | --------------- | ---------------------------------------- |
| 欠品削減   | 人気商品の欠品回数           | 月間50%削減     | 対象商品を定義し、欠品発生を日次で記録   |
| 在庫適正化 | 廃棄・値引き・長期滞留の件数 | 半年で20%削減   | 廃棄、値引き、滞留を記録し、月次で集計   |
| つけ管理   | つけ残高の確認時間           | 1分以内         | 顧客検索から残高確認までの所要時間を観察 |
| 継続運用   | 店主が記録を継続できた日数   | 営業日の90%以上 | 販売、在庫、つけの記録有無を日次で確認   |

参考指標として、記録対象商品の数、常連顧客の登録件数、家族による店番時の利用回数を継続的に確認する。

## 6. 前提条件

本プロジェクトの策定および活用にあたっては、以下を前提とする。

- 対象は単一店舗（きぬや）の店頭販売、店頭在庫、常連顧客のつけ管理に限定すること。
- 初期リリースでは、EC 連携、複数店舗管理、本格会計連携、仕入れ先システム連携は対象外とすること。
- 操作端末は店内のタブレット 1 台を想定すること。
- 店舗の接客手順を大きく変えず、現行運用に近い入力手順を優先すること。
- 店主が最終的な業務判断を行い、システムは記録と確認を支援する位置づけとすること。
- オフライン時の運用方針は _UNDECIDED_: 初期リリース計画で決定すること。
