# 成功基準と受入条件

## 1. 判定対象と適用範囲

本書は、SpecDojo プロジェクトの初期整備完了、および本格実行開始または初回公開に向けた GO / Not GO 判断に用いる成功基準と受入条件を定義する。

判定対象は、SpecDojo の文書体系、作成ルール、サンプル、AI 向け指示テンプレート、プロジェクト管理文書、公開準備の状態とする。特定業務システムの実装品質や利用プロジェクト側の成果は、本書の受入対象外とする。

## 2. 成功基準

| ID    | 条件                                             | 判定基準                                                                             | 測定方法                                   | 判定時期           | 責任者            |
| ----- | ------------------------------------------------ | ------------------------------------------------------------------------------------ | ------------------------------------------ | ------------------ | ----------------- |
| SC-01 | 主要文書体系が定義されている                     | standards、guidelines、rulebooks、instructions、samples の配置方針と役割が説明できる | 成果物カタログ、構成ガイド、関連文書の確認 | GO / Not GO 判断前 | `PO`, `ARC`       |
| SC-02 | 主要成果物の作成ルールとサンプルが整備されている | 優先度の高い文書種別に rulebook、sample、instruction が存在する                      | 成果物カタログと実ファイルの照合           | 初回公開前         | `ARC`, `QE`       |
| SC-03 | 人と AI Agent が利用できる粒度になっている       | 指示テンプレート、サンプル、関連リンクにより、次の成果物作成に使える                 | AI 利用レビュー、サンプル確認              | 初回公開前         | `BA`, `QE`        |
| SC-04 | プロジェクト管理の最小基盤がある                 | スコープ、成功基準、前提・制約、管理計画、組織、RACI、WBS、Schedule が定義されている | プロジェクト定義・管理文書の確認           | GO / Not GO 判断前 | `PO`, `QE`        |
| SC-05 | 公開・改善・再利用しやすい状態である             | README、LICENSE、公開範囲、Issue / PR 方針の未決事項が整理されている                 | 公開前レビュー、未決事項確認               | 初回公開前         | `PO`, `ARC`, `QE` |
| SC-06 | 品質確認が可能である                             | lint、レビュー観点、課題・リスク・変更管理の導線がある                               | 品質管理計画、lint 結果、登録簿の確認      | 主要リリース前     | `QE`              |

## 3. 受入条件

| ID    | 種別     | 条件                                           | 合格基準                                                                                     | 証跡                                                     | 承認者 |
| ----- | -------- | ---------------------------------------------- | -------------------------------------------------------------------------------------------- | -------------------------------------------------------- | ------ |
| AC-01 | 文書体系 | 主要ディレクトリと成果物分類が説明可能である   | 構成ガイドと成果物カタログが矛盾していない                                                   | dct-index、docs-structure-guide                          | `PO`   |
| AC-02 | ルール   | 主要成果物に作成ルールがある                   | 対象 rulebook が存在し、必須構成を示している                                                 | rulebooks 配下の対象ファイル                             | `PO`   |
| AC-03 | サンプル | 利用者が成果物の完成イメージを確認できる       | 対象 sample が存在し、rulebook と大きく矛盾しない                                            | samples 配下の対象ファイル                               | `PO`   |
| AC-04 | AI 指示  | AI Agent に作成依頼できる入力形式がある        | 対象 instruction が存在し、入力・制約・出力条件を示している                                  | instructions 配下の対象ファイル                          | `PO`   |
| AC-05 | 管理文書 | GO / Not GO 判断に必要な管理文書がそろっている | スコープ、成功基準、前提・制約、管理計画、品質、コミュニケーション、組織、RACI が draft 以上 | project-definition / management-plan / organization 配下 | `PO`   |
| AC-06 | 品質     | 重大な文書構造不備が残っていない               | `npm run -s lint:md` が成功し、重大レビュー指摘が 0 件                                       | lint 結果、レビュー記録                                  | `PO`   |
| AC-07 | 公開適性 | 公開不適切情報を含まない                       | 個人情報、非公開情報、機密情報が含まれていない                                               | 公開前レビュー結果                                       | `PO`   |

## 4. 判定手順と証跡

1. `PO` は判定対象の成果物一覧を成果物カタログで確認する。
2. `ARC` は文書体系、配置、ID、派生関係の整合性を確認する。
3. `QE` は lint、レビュー指摘、文書間矛盾、公開適性を確認する。
4. `BA` は利用者視点と AI 利用可能性を確認する。
5. 判定結果、未決事項、保留条件はプロジェクト登録簿または決定記録に残す。
6. `PO` が GO / Not GO または公開可否を判断する。

## 5. 例外条件と未解決事項

| 論点                 | 扱い                           | 解決期限 / 判断タイミング            | 責任者           |
| -------------------- | ------------------------------ | ------------------------------------ | ---------------- |
| 全文書種別の完成     | 初期公開では必須としない       | 優先度と成果物カタログで段階的に判断 | `PO`, `ARC`      |
| 外部貢献ルール       | 初回公開前に最小導線を整備する | 初回公開前                           | `PO`, `BA`, `QE` |
| 定量的な利用実績指標 | 初回公開後に測定を開始する     | 初回公開後                           | `PO`             |
| ライセンス方針       | 公開前までに確定する           | 初回公開前                           | `PO`             |
