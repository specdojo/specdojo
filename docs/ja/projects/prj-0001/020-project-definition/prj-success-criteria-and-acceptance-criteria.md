# 成功基準と受入条件

## 1. 判定対象と適用範囲

本書は、[[prj-0001:prj-scope|プロジェクトスコープ]]で定めた初期対象について、SpecDojo プロジェクトの初期整備完了、および本格実行開始または初回公開に向けた GO / Not GO 判断に用いる成功基準と受入条件を定義する。

判定対象は、文書体系、成果物に紐づく参考資料、プロジェクト管理文書、公開・再利用のための準備状態とする。特定業務システムの実装品質、利用プロジェクト側の成果、初回公開後の利用実績は受入対象外とする。利用実績は公開後の継続改善で測定する。

| 業務価値 ID | 業務価値                                                              | 判定対象との対応                                          |
| ----------- | --------------------------------------------------------------------- | --------------------------------------------------------- |
| BV-01       | 市民、専門家、行政、開発者、AI Agent が同じ仕様と判断根拠を参照できる | 文書体系、プロジェクト定義・管理文書、利用可能な参照導線  |
| BV-02       | 複数プロジェクトで再利用できる文書体系と共通の型を使える              | rulebook、recipe、sample、template の整合した参考資料一式 |
| BV-03       | 公開・再利用を妨げずに文書を作成・レビュー・改善できる                | 公開範囲、判定証跡、未決事項の明示                        |

## 4. 判定手順と証跡

1. `BA` は、判定対象と業務価値・利用者・利用場面との対応を確認する。
2. `ARC` は、文書構造、ID、参照関係、技術的受入条件の確認範囲を確認する。
3. `QE` は、各条件の証跡、lint 結果、公開適性、文書間矛盾を確認する。
4. 否決条件がある場合は、是正内容、再確認する受入条件、再判定の予定を記録する。
5. 判定結果、未決事項、保留条件は決定記録またはプロジェクト登録簿に残す。
6. `PO` が確認結果と証跡をもとに GO / Not GO または公開可否を判断する。

## 5. 例外条件と未解決事項

| 論点                 | 扱い                           | 解決期限 / 判断タイミング              | 責任者           |
| -------------------- | ------------------------------ | -------------------------------------- | ---------------- |
| 全文書種別の完成     | 初期公開では必須としない       | 優先度とスコープに照らして段階的に判断 | `PO`, `ARC`      |
| 外部貢献ルール       | 初回公開前に最小導線を整備する | 初回公開前                             | `PO`, `BA`, `QE` |
| 定量的な利用実績指標 | 初回公開後に測定を開始する     | 初回公開後                             | `PO`             |
| ライセンス方針       | 公開前までに確定する           | 初回公開前                             | `PO`             |
