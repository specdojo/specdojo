# プロジェクト憲章: SpecDojo

本書は、SpecDojo プロジェクトの立ち上げを正式に認可し、詳細計画の策定と初期準備を進める権限を委譲するための憲章である。PO は、目的、優先順位、公開方針、主要な合意を最終判断する人間の責任者として、本書に記載した範囲を承認または差し戻す。

## 1. 本書の目的

- SpecDojo を、オープンなドキュメントフレームワーク整備プロジェクトとして立ち上げることを認可する。
- プロジェクトマネージャーに、初期公開へ向けた詳細計画、成果物体系、管理文書、公開準備を進める権限を委譲する。
- 本書の承認は、プロダクト成果物の本格実行や外部公開そのものを承認するものではない。外部公開、ライセンス、主要スコープ変更、GO / Not GO 判断は、PO が別途判断する。

## 3. プロジェクトの目的

SpecDojo は、人と AI Agent が同じ仕様と判断根拠を参照し、プロジェクトの立ち上げ、改善、運用を支援できる共有知識基盤を整備する。目的は、仕様駆動プロジェクト運営を支える文書体系、規約、テンプレート、サンプル、ツール、AI Agent 向け指示を、オープンソースとして利用・改善できる形にすることである。

認可判断における期待効果は次のとおりである。

| 観点 | 認可判断で確認する期待効果 | 根拠 |
| --- | --- | --- |
| 仕様共有 | 人間と AI Agent が同じ仕様体系を参照して作業できる | [[prj-0001:prj-overview\|プロジェクト概要]] |
| 透明性 | 判断根拠、仕様、変更履歴を公開可能な形で残せる | [[prj-0001:prj-overview\|プロジェクト概要]] |
| 継続性 | 担当者交代や長期運用時も、知識と判断を引き継げる | [[prj-0001:prj-overview\|プロジェクト概要]] |
| 協働性 | 市民、専門家、行政、開発者、AI Agent が協働しやすい共通の型を提供できる | [[prj-0001:prj-overview\|プロジェクト概要]] |

## 8. 権限委譲

本書の承認により、プロジェクトマネージャーは次の範囲で初期準備を進める権限を持つ。

| 項目 | 決裁者 | 実行責任者 | 協議先 | 証跡 |
| --- | --- | --- | --- | --- |
| 立ち上げ認可 | `PO` | `PM` | `BA` | 本書の承認履歴 |
| 詳細計画策定 | `PO` | `PM` | `BA` / `QE` | `pm-plan`、Issue、Pull Request |
| 成果物体系の整理 | `PO` | `PM` | `BA` / `ARC` / `QE` | 成果物カタログ、Pull Request |
| AI Agent による草案作成・整合確認 | `PO` | `PM` | `BA` | exec plan、生成結果、レビュー記録 |
| 公開準備の検討 | `PO` | `PM` | 将来利用者・貢献者の想定観点 | README、Issue、決定記録 |

次の事項は、委譲範囲に含めず PO の承認を必要とする。

- プロジェクト目的の変更
- 主要スコープの大幅な変更
- 初期公開範囲、公開可否、公開日、ライセンス方針の確定
- 主要成果物群の追加または削除
- 予算枠の新設、増額、または外部支出の承認
- 各 GO / Not GO 判断
- プロジェクト中止または大幅な再計画の判断

## 9. 主要前提・制約

| 区分 | 内容 | 詳細化先 |
| --- | --- | --- |
| 前提 | SpecDojo は人間の判断や責任を代替せず、人と AI Agent が協働するための基盤として整備する | `prj-assumptions-constraints-dependencies` |
| 前提 | AI Agent は意思決定者ではなく、課題、目的、選択肢、判断根拠の整理と作業支援を担う | `prj-assumptions-constraints-dependencies` |
| 前提 | 文書は、人間の理解容易性と AI Agent の解釈容易性を両立させる | `prj-assumptions-constraints-dependencies` |
| 制約 | 特定の開発手法、言語、基盤、生成 AI 製品に過度に依存しない | `prj-assumptions-constraints-dependencies` |
| 制約 | 公開リポジトリで扱える情報に限定し、個人情報、秘密情報、非公開の組織情報を含めない | `prj-assumptions-constraints-dependencies` |
| 制約 | 予算枠は `_UNDECIDED_:` 初期公開計画または PO 承認時に確定する。本書では、追加支出を伴わない文書整備と公開準備の立ち上げのみを認可する | 未決事項 |

## 11. 本格実行開始の GO / Not GO 判断

本書の承認後、次の観点が確認できるまで、本格実行開始と外部公開の最終判断は保留する。

| 判断観点 | 確認内容 | 記録先 |
| --- | --- | --- |
| 目的整合 | 成果物体系、公開対象、優先順位が SpecDojo の目的と矛盾していない | `pm-plan` / 決定記録 |
| 公開適性 | 公開してよい情報だけで構成され、ライセンス方針と公開範囲が PO により承認されている | 決定記録 / Pull Request |
| 予算枠 | 追加支出、外部サービス利用、運用負担の上限が承認されている | 決定記録 |
| 体制 | PO、PM、BA、レビュー担当の責任範囲が確認できる | `pm-organization` / `pm-raci` |
| 品質 | 初期公開に必要な rulebook、recipe、sample、template、検証方針が整っている | `pm-quality-management-plan` |
