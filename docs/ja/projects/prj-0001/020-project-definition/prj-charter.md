# プロジェクト憲章: SpecDojo

本書は、SpecDojo をオープンなドキュメントフレームワーク整備プロジェクトとして立ち上げることを正式に認可し、プロジェクトマネージャーに詳細計画の策定と追加支出を伴わない初期準備を進める権限を委譲するための憲章である。PO は、目的、優先順位、公開方針、主要な合意を最終判断する人間の責任者として、本書に記載した範囲を承認または差し戻す。本書の承認は、本格実行開始、外部公開、追加支出を承認するものではなく、これらは PO が別途判断する。

## 1. 認可対象

| 項目            | 内容                                                                                                       |
| --------------- | ---------------------------------------------------------------------------------------------------------- |
| プロジェクト名  | SpecDojo                                                                                                   |
| プロジェクト ID | `prj-0001`                                                                                                 |
| 認可対象        | プロジェクト立ち上げ、詳細計画策定、追加支出を伴わない初期公開準備の検討                                   |
| 認可しない範囲  | 外部公開の最終判断、ライセンス確定、主要スコープ変更、本格実行開始、予算枠の確定または外部支出             |
| 直接根拠        | [[prj-0001:prj-overview\|プロジェクト概要]]、[[prj-0001:prj-stakeholder-register\|ステークホルダー登録簿]] |
| 承認責任        | `PO`                                                                                                       |
| 認可条件        | 承認日と証跡リンクを記録し、本書が本格実行開始、外部公開、追加支出を承認しないことを確認する               |

## 2. プロジェクトの目的

SpecDojo は、人と AI Agent が同じ仕様と判断根拠を参照し、プロジェクトの立ち上げ、改善、運用を支援できる共有知識基盤を、オープンソースとして整備する。背景、必要性、目的の詳細は [[prj-0001:prj-overview|プロジェクト概要]] を正とする。

認可判断における期待効果は次の観点で確認する。数値目標、測定方法、中長期指標は [[prj-0001:prj-overview|プロジェクト概要]] の期待効果を正とし、本書では再掲しない。

| 観点     | 認可判断で確認する期待効果                                              |
| -------- | ----------------------------------------------------------------------- |
| 仕様共有 | 人間と AI Agent が同じ仕様体系を参照して作業できる                      |
| 透明性   | 判断根拠、仕様、変更履歴を公開可能な形で残せる                          |
| 継続性   | 担当者交代や長期運用時も、知識と判断を引き継げる                        |
| 協働性   | 市民、専門家、行政、開発者、AI Agent が協働しやすい共通の型を提供できる |

成功判定の観点と受入条件は [[prj-0001:prj-success-criteria-and-acceptance-criteria|成功基準と受入条件]] を正とする。

## 5. 権限委譲

本書が PO に承認された場合、プロジェクトマネージャーは次の範囲で初期準備を進める権限を持つ。

| 項目                                | 決裁者 | 実行責任者 | 協議先                        | 証跡                                                            |
| ----------------------------------- | ------ | ---------- | ----------------------------- | --------------------------------------------------------------- |
| 立ち上げ認可                        | `PO`   | `PM`       | `BA`                          | 本書の承認履歴                                                  |
| 詳細計画策定                        | `PO`   | `PM`       | `BA`                          | [[prj-0001:pm-plan\|プロジェクト管理計画]]、Issue、Pull Request |
| 成果物体系の整理                    | `PO`   | `PM`       | `BA` / `_TODO_: レビュー担当` | 成果物カタログ、Pull Request                                    |
| AI Agent による草案作成・整合確認   | `PO`   | `PM`       | `BA`                          | exec plan、生成結果、レビュー記録                               |
| 本格実行開始の GO / Not GO 判断準備 | `PO`   | `PM`       | `BA` / `_TODO_: レビュー担当` | GO / Not GO 判断の決定記録                                      |
| 公開準備の検討                      | `PO`   | `PM`       | 将来利用者・貢献者の想定観点  | README、Issue、決定記録                                         |

次の事項は、委譲範囲に含めず PO の承認を必要とする。

- プロジェクト目的の変更
- 主要スコープの大幅な変更
- 初期公開範囲、公開可否、公開日、ライセンス方針の確定
- 主要成果物群の追加または削除
- 予算枠の新設、増額、または外部支出の承認
- 各 GO / Not GO 判断
- プロジェクト中止または大幅な再計画の判断

## 6. 主要前提・制約

前提・制約の全量は [[prj-0001:prj-scope|プロジェクトスコープ]]（`対象システム`、`スコープ外`、`境界の判断基準`）を正とし、成立条件の監視と未確定の依存は [[prj-0001:prj-assumptions-constraints-dependencies|前提・制約・依存関係]] が扱う。本書では認可判断に直結する事項だけを示す。

| 区分 | 内容                                                                                                                                   | 詳細化先                                     |
| ---- | -------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------- |
| 前提 | SpecDojo は人間の判断や責任を代替せず、人と AI Agent が協働するための基盤として整備する                                                | [[prj-0001:prj-scope\|プロジェクトスコープ]] |
| 制約 | 公開リポジトリで扱える情報に限定し、個人情報、秘密情報、非公開の組織情報を含めない                                                     | [[prj-0001:prj-scope\|プロジェクトスコープ]] |
| 制約 | 予算枠は `_UNDECIDED_:` 初期公開計画または PO 承認時に確定する。本書では、追加支出を伴わない文書整備と公開準備の立ち上げのみを認可する | 未決事項                                     |
| 制約 | 外部サービス利用、有償ツール利用、運用負担を伴う公開準備は、予算枠と公開方針が PO に承認されるまで実施しない                           | 未決事項                                     |

## 7. 本格実行開始の GO / Not GO 判断

本書の承認後、次の観点が確認できるまで、本格実行開始と外部公開の最終判断は保留する。

| 判断観点 | 確認内容                                                                           | 記録先                                                              |
| -------- | ---------------------------------------------------------------------------------- | ------------------------------------------------------------------- |
| 目的整合 | 成果物体系、公開対象、優先順位が SpecDojo の目的と矛盾していない                   | [[prj-0001:pm-plan\|プロジェクト管理計画]] / 決定記録               |
| 公開適性 | 公開してよい情報だけで構成され、ライセンス方針と公開範囲が PO により承認されている | 決定記録 / Pull Request                                             |
| 予算枠   | 追加支出、外部サービス利用、運用負担の上限が承認されている                         | 決定記録                                                            |
| 体制     | PO、PM、BA、レビュー担当の責任範囲が確認できる                                     | [[prj-0001:pm-organization\|組織定義]] / [[prj-0001:pm-raci\|RACI]] |
| 品質     | 初期公開に必要な rulebook、recipe、sample、template、検証方針が整っている          | [[prj-0001:pm-quality-management-plan\|品質管理計画]]               |
