# プロジェクト憲章: SpecDojo

## 1. 本書の目的

本書は、SpecDojo プロジェクトの立ち上げを正式に認可し、プロジェクトマネージャーに詳細計画の策定および初期準備を進める権限を付与するための文書である。

本書は、プロジェクト定義文書群の詳細内容を再掲しない。
背景、目的、スコープ、成功基準、前提・制約、ステークホルダーの詳細は、各詳細文書を正とする。

なお、本書の承認は本格実行の開始を意味しない。
GO / Not GO 判断は [ドキュメント構成ガイド](../../../specdojo/guidelines/docs-structure-guide.md) の作成順・検討順に従い、`A. 立ち上げ`、`B. プロジェクト定義`、`C. プロジェクトマネジメント`、`D. プロダクト変更`、`E. プロダクト成果物` の進行に応じて段階的に行う。

## 2. 認可対象

本書により、次のプロジェクトについて、立ち上げ、詳細計画の策定、および初期準備を認可する。

| -------------------- | ------------------------------------------------------------ |
| プロジェクト名       | SpecDojo                                                     |
| プロジェクトID       | prj-0001                                                     |
| 認可対象             | プロジェクトの立ち上げ、詳細計画の策定、初期準備             |
| 後続ゲート           | 本書では認可しない。各 GO / Not GO 判断で別途確認する        |
| プロジェクト概要     | [prj-overview.md](./prj-overview.md)                         |
| 初期ステークホルダー | [prj-stakeholder-register.md](./prj-stakeholder-register.md) |

## 3. プロジェクト定義文書群との関係

`020-project-definition` 配下の文書群は、SpecDojo プロジェクトの初期定義パッケージである。
本書は、そのうち立ち上げ認可と権限委譲を記録するプロジェクト憲章として位置づける。
文書群の作成順・検討順は [ドキュメント構成ガイド](../../../specdojo/guidelines/docs-structure-guide.md) を正とする。

| 領域                         | 正とする文書                                                                                     | 本書での扱い                   |
| ---------------------------- | ------------------------------------------------------------------------------------------------ | ------------------------------ |
| 背景、目的、必要性、期待効果 | [prj-overview.md](./prj-overview.md)                                                             | 認可判断の根拠として参照する   |
| 課題と解決アプローチ         | [prj-issues-and-approach.md](./prj-issues-and-approach.md)                                       | 詳細化文書として参照する       |
| 代替案比較                   | [prj-comparison-of-alternatives.md](./prj-comparison-of-alternatives.md)                         | 採択理由の根拠として参照する   |
| スコープ                     | [prj-scope.md](./prj-scope.md)                                                                   | 詳細化文書として参照する       |
| 成功基準、完了定義、受入条件 | [prj-success-criteria-and-acceptance-criteria.md](./prj-success-criteria-and-acceptance-criteria.md) | 詳細化文書として参照する       |
| 前提、制約、依存関係         | [prj-assumptions-constraints-dependencies.md](./prj-assumptions-constraints-dependencies.md)     | 詳細化文書として参照する       |
| ステークホルダー             | [prj-stakeholder-register.md](./prj-stakeholder-register.md)                                     | 初期関係者の根拠として参照する |

上記文書の内容が更新された場合でも、本書の認可対象、権限委譲、承認条件に影響しない限り、本書の改訂は不要とする。
本書の認可範囲または承認条件を変更する場合は、本書を改訂し、必要に応じて再承認する。

## 4. 権限委譲

本憲章の承認により、プロジェクトマネージャーは、次の権限を持つ。

- プロジェクト定義文書を作成、更新、整理する
- 成果物カタログを作成、更新、整理する
- プロジェクトマネジメント文書を作成、更新、整理する
- 組織体制、ロール、責任分担を整理する
- 作成対象文書の優先順位を調整する
- AI Agent を活用して文書の草案作成、レビュー、改善を進める
- 初期成果物の作成、更新、レビューを進める
- リポジトリ構成、README、ライセンス、貢献導線の整備を検討する

ただし、次の事項はスポンサーまたはプロジェクトオーナーの承認を必要とする。

- プロジェクト目的の変更
- 認可対象の変更
- 主要スコープの大幅な変更
- 主要成果物群の追加または削除
- オープンソース公開方針の変更
- ライセンス方針の変更
- 各ゲートの GO / Not GO 判断
- プロジェクト中止または大幅な再計画の判断

## 7. GO / Not GO 判断との関係

GO / Not GO 判断は、[ドキュメント構成ガイド](../../../specdojo/guidelines/docs-structure-guide.md) の三段階ゲートに従う。
本書の承認は、`A. 立ち上げ` の一部として、`B. プロジェクト定義` および `C. プロジェクトマネジメント` の作成・詳細化へ進むことを認可するものである。

| ゲート | 判断タイミング | 判断内容 | 本書での扱い |
| ------ | -------------- | -------- | ------------ |
| 第1ゲート | `A. 立ち上げ`、`B. プロジェクト定義`、`C. プロジェクトマネジメント` が完了した時点 | TO-BE が明確化されたか | 判断結果は本書では確定しない |
| 第2ゲート | `D. プロダクト変更` が完了した時点 | TO-BE の実現性が明確化されたか | 判断結果は本書では確定しない |
| 第3ゲート | `E. プロダクト成果物` が完了した時点 | 負荷・期間が明確化されたか | 判断結果は本書では確定しない |

各ゲートの判断日、判定者、条件、証跡は、該当するプロジェクト管理文書、プロジェクト登録簿、または意思決定ログで記録する。
