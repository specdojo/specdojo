# 前提・制約・依存関係

本書は、[[prj-0001:prj-scope|プロジェクトスコープ]] の実行に影響する成立条件、守るべき境界、外部または先行成果物への依存を整理する。スコープが定義済みの境界（`対象システム`、`スコープ外`、`境界の判断基準`）は再掲せず、スコープを正とする。本書は、スコープに現れない成立条件と、未確定の依存の所在だけを扱う。

## 1. 前提条件

初期公開の前提（人と AI Agent の協働、主要文書体系の優先、成果物の選択制）は、[[prj-0001:prj-scope|プロジェクトスコープ]] の `スコープ外` と `境界の判断基準` で定義済みであり、本書では再掲しない。スコープに追加する前提条件は現時点でない。前提を追加する場合は、スコープと重複しないことを確認して本章の表に追記する。

## 2. 制約事項

公開・再利用・技術境界に関する制約は [[prj-0001:prj-scope|プロジェクトスコープ]] の `対象システム` と `スコープ外` を正とする。本書では、スコープに現れない文書体系の運用制約のみを扱う。

| ------- | -------------------------------------------------------------------------------------------------- | ---------------------------- | -------------------------------------------------------------- | --------------------------------------------------------------------------- | ----------------------------------------------------------- | ------ | ------------------------------------------------------ |
| ACD-C04 | 成果物は Markdown、frontmatter、ID、配置、命名、リンクを一貫して扱い、個別の技術設計を代替しない。 | 文書フレームワークと参考資料 | 文書間の参照や再利用が困難になり、設計詳細との責務が混在する。 | frontmatter、ID、ファイル名、見出し、リンクが規約と整合することを確認する。 | 構成情報が欠落した、または API・DB 等の設計詳細が混入した。 | `ARC`  | 文書構造を是正し、詳細は対応する後続成果物へ委譲する。 |

## 3. 依存関係

| ID      | 依存先・条件                                     | 必要となる理由                                                                               | 受領・成立条件                                     | 変化のトリガー                               | 所有者 | 対応方針                                                   |
| ------- | ------------------------------------------------ | -------------------------------------------------------------------------------------------- | -------------------------------------------------- | -------------------------------------------- | ------ | ---------------------------------------------------------- |
| ACD-D01 | [[prj-0001:prj-scope\|プロジェクトスコープ]]     | 対象業務、対象システム、対象外、境界判断を前提として、技術制約と依存の適用範囲を定めるため。 | 本書の各項目がスコープ内・スコープ外と矛盾しない。 | スコープまたは境界判断基準が変更された。     | `ARC`  | 本書の該当項目を再評価し、影響がある未確定事項を記録する。 |
| ACD-D02 | rulebook / recipe / sample / template の参照一式 | 文書体系の利用者と AI Agent が、構造、作り方、完成例、雛形をそれぞれ参照できるため。         | 4 種別が同じ章構成・必須項目・禁止事項に整合する。 | いずれかの参考資料が追加・変更・廃止された。 | `ARC`  | rulebook を正として相互参照と構造を更新する。              |

公開方針に関する未確定の依存（初期公開の範囲・ライセンス・貢献導線、公開先・変更提案の受付方法）は、[[prj-0001:pjr-index|プロジェクト登録簿]] の `PJR-0120`、`PJR-0121` で決定待ちの事項として管理する。決定後、本書に影響がある場合は該当項目へ反映する。

## 4. 影響評価と対応方針

- スコープ由来の境界が変化した場合は、[[prj-0001:prj-scope|プロジェクトスコープ]] の `スコープ変更方針` に従って影響を評価し、`PO` が判断する。
- 本書の制約（ACD-C04）に抵触した場合は、`ARC` が文書構造を是正し、詳細は対応する後続成果物へ委譲する。
- 依存先（スコープ、参考資料一式、登録簿の決定事項）が変化した場合は、`ARC` が本書と参考資料の整合を確認し、未判断の影響は `_TODO_` として残す。

## 5. 監視・変更管理

- 本書は、[[prj-0001:prj-scope|プロジェクトスコープ]]、参考資料一式、または登録簿の決定事項（`PJR-0120`、`PJR-0121`）が変わったときに見直す。
- 変更時は、項目 ID、変化の内容、影響範囲、判断者、対応状況を [[prj-0001:pjr-index|プロジェクト登録簿]] に記録する。
- `ARC` は文書構造、配置、命名、技術制約、参照資料の整合を確認する。公開可否やスコープ変更の最終判断は、人間の責任者が行う。
