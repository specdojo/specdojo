# 組織定義

本書は、本プロジェクトにおけるロール・メンバー構成の方針と設計根拠を定義する。[[prj-0001:prj-overview|プロジェクト概要]] が掲げる、人と AI Agent が共有する仕様体系、オープンソースとして公開可能な共通基盤、継続的な改善を支えるため、組織構成の最終判断を人間の PO に集約する。

## 1. 基本方針

- 本プロジェクトは、人と AI Agent の協働を前提とした個人・小規模運用から開始する。初期公開では、全成果物の完成ではなく、主要文書体系、ルール、サンプル、AI Agent 向け指示、管理基盤が参照可能な状態になることを優先する。
- 組織設計は、公開可能な文書フレームワークを整備する目的、特定製品・個別組織に依存しないスコープ、PO による公開判断という制約に従う。
- 初期運用の兼務構成は、限られた体制で主要文書体系を整えるための暫定的な設計とする。判断責任は PO に集約し、実行支援や機械的な整合確認は AI Agent が担える範囲に限定する。
- 人間の PO は、目的・スコープ・優先順位・公開方針との整合、未決事項、影響範囲、公開してよい情報だけで構成されているかを確認して、組織構成を承認、保留、または差し戻す。
- PO が承認できない未決事項が残る場合は、本書に責務語彙や member 割り当てを先行して書き込まず、判断時点または判断者を明示して保留する。影響範囲は見直し条件に沿って確認し、Role code の正本と実行主体の正本へ分けて反映する。
- AI Agent は、草案作成、差分整理、整合確認、機械的更新を支援する。最終判断、公開可否、説明責任は人間の PO が担う。
- `pm-roles.yaml` は Role code の語彙を管理し、`pm-members.yaml` は実行主体と兼務割り当てを管理する。本書は両者の一覧を複製せず、組織設計の方針と採用根拠だけを扱う。

## 2. 採用ロールと owner 語彙

`pm-roles.yaml` に定義した Role code を、本プロジェクトで使用可能な責務語彙として採用する。ここでの採用は、Schedule の `owner` や RACI の列として使える語彙を定めることであり、すべての Role に専任 member を置くことではない。

初期運用では、人間の最終判断主体が必要な責務を兼務できる構成とする。実行時には、タスクの性質に近い Role code を `owner` に置き、実際の実行主体と対応 Role code は [[prj-0001:pm-members|メンバー定義]] で管理する。

Schedule の `owner` に使える値は、[[prj-0001:pm-roles|ロール定義]] の `roles[].code` に存在する Role code に限定する。member nickname、agent 名、個人名、未定義の独自 Role code は `owner` として扱わない。新しい責務語彙が必要な場合は、Schedule 側で独自値を先行利用せず、PO が目的・公開方針・再利用性への影響を確認したうえで `pm-roles.yaml` の更新要否を判断する。

## 5. 禁止事項

- Role code の一覧、具体的な member、兼務の割り当てを本書に複製しない。
- 個人名、member nickname、agent 名を Schedule の `owner` として扱わない。
- [[prj-0001:pm-roles|ロール定義]] にない Role code を Schedule の `owner` として扱わない。
- AI Agent に最終承認、公開可否、説明責任を委ねない。
- 公開文書に不要な個人情報、連絡先、非公開組織情報、アクセス情報を記載しない。
