# 組織定義

本書は、本プロジェクトにおけるロール・メンバー構成の方針と設計根拠を定義する。[[prj-overview|プロジェクト概要]] が掲げる、人と AI Agent が共有する仕様体系、公開可能な共通基盤、継続的な改善を支えるため、組織構成の最終判断を人間の PO に集約する。

## 1. 基本方針

- 本プロジェクトは、人と AI Agent の協働を前提とした個人・小規模運用から開始する。初期公開では、全成果物の完成ではなく、主要文書体系と参考資料の成立を優先する。
- 組織設計は、公開可能な文書フレームワークを整備する目的、特定製品・個別組織に依存しないスコープ、PO による公開判断という制約に従う。
- 人間の PO は、目的・スコープ・優先順位・公開方針との整合、未決事項、影響範囲を確認して、組織構成を承認、保留、または差し戻す。
- AI Agent は、草案作成、差分整理、整合確認、機械的更新を支援する。最終判断、公開可否、説明責任は人間の PO が担う。
- Role code の語彙、実行主体、具体的な兼務割り当ては本書に複製せず、関連ドキュメントを正本とする。

## 2. 採用ロールと owner 語彙

`pm-roles.yaml` に定義した Role code を、本プロジェクトで使用可能な責務語彙として採用する。ここでの採用は、Schedule の `owner` や RACI の列として使える語彙を定めることであり、すべての Role に専任 member を置くことではない。

初期運用では、人間の最終判断主体が必要な責務を兼務できる構成とする。実行時には、タスクの性質に近い Role code を `owner` に置き、実際の実行主体と対応 Role code は [[prj-0001:pm-members|メンバー定義]] で管理する。

Schedule の `owner` に使える値は、[[prj-0001:pm-roles|ロール定義]] の `roles[].code` に存在する Role code に限定する。member nickname、agent 名、個人名、未定義の独自 Role code は `owner` として扱わない。

## 4. 見直し条件

| 更新トリガー | PO が確認する影響範囲 | 更新対象 |
| --- | --- | --- |
| 目的、スコープ、優先順位、公開方針が変更された | 採用する責務語彙、兼務継続の妥当性、公開判断の集約先 | 本書、[[prj-0001:pm-roles\|ロール定義]]、必要に応じて [[prj-0001:pm-members\|メンバー定義]] |
| 継続的に参加する人または AI Agent が増えた | 専任化が必要な責務、実行主体と Role code の対応、RACI の必要性 | [[prj-0001:pm-members\|メンバー定義]]、必要に応じて [[prj-0001:pm-raci\|RACI]] |
| 高優先度タスク、公開判断、運用対応が PO 兼務で継続的に滞留した | PM、OPS、QE などの責務を分離する必要性と承認責任の残し方 | 本書、[[prj-0001:pm-roles\|ロール定義]]、[[prj-0001:pm-members\|メンバー定義]] |
| 新しい Role code が必要になった | 標準 Role code で代替できるか、独自 Role が公開文書の再利用性を損なわないか | 本書、[[prj-0001:pm-roles\|ロール定義]]、Schedule、必要に応じて [[prj-0001:pm-raci\|RACI]] |
| 公開対象に個人情報、非公開情報、アクセス情報が含まれる可能性が出た | 公開可否、匿名化または削除の要否、説明責任の所在 | 本書、[[prj-0001:pm-members\|メンバー定義]]、公開前レビュー対象 |

## 5. 禁止事項

- Role code の一覧、具体的な member、兼務の割り当てを本書に複製しない。
- 個人名、member nickname、agent 名を Schedule の `owner` として扱わない。
- [[prj-0001:pm-roles|ロール定義]] にない Role code を Schedule の `owner` として扱わない。
- AI Agent に最終承認、公開可否、説明責任を委ねない。
- 公開文書に不要な個人情報、連絡先、非公開組織情報、アクセス情報を記載しない。
