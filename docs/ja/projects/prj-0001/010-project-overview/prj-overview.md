# プロジェクト概要: SpecDojo Handbook

## 1. 背景

生成AIおよび各種Agentの発展により、システムの企画、設計、実装、テスト、運用・保守に至るまで、自動化可能な業務領域が拡大している。
こうした環境において、AIが適切にタスクを実行するためには、要件、仕様、設計、制約条件等を、曖昧さなく構造的に記述し、共有することが不可欠である。

従来、開発文書は主として人間同士の理解と合意形成のためのものであった。しかし今後は、人間に加えてAIも読み手・実行主体となることから、ドキュメントには可読性だけでなく、解釈可能性、再利用性、実行可能性が求められる。

本プロジェクトでは、このようなドキュメントを接点として、人とAIが協働しながらシステムを構築・改修する開発スタイルを**仕様駆動開発**と位置付ける。

## 2. 目的

仕様駆動開発のための、オープンかつライフサイクル全体を対象としたドキュメントフレームワーク **SpecDojo** を整備する。
SpecDojo では、システムの構築・改修に必要な文書群を体系化し、記述規約、テンプレート、サンプル、AI向け指示テンプレート、補助ツールを通じて、企画から運用・保守までを一貫して支援する。

あわせて、以下を満たすドキュメンテーションの実践基盤を確立する。

- AIが扱いやすいこと
- 人が理解しやすいこと
- 継続的に記述・更新できること

これにより、開発の生産性、品質、継続性の向上を図る。

## 3. 必要性

システム開発の上流工程におけるドキュメンテーションは、これまで担当者の経験や力量に依存しやすく、標準化・再利用が進みにくかった。
その結果、要件や設計意図の伝達不足による認識齟齬、手戻り、属人化、ブラックボックス化が多くのプロジェクトで課題となっている。

さらに、AIが実装、テスト、運用等の作業を担うようになったことで、ドキュメントには従来の人間向け記述に加え、AIが構造的に解釈し、タスク実行に利用できることが求められている。
一方で、どの文書を、どの粒度で、どのような構造・規約で整備すべきかについては、現時点で共通基盤が十分に存在しない。

このため、仕様駆動開発に適したドキュメンテーションを再定義し、体系化し、実践可能な形で提示することが必要である。

## 4. 期待効果

SpecDojo Handbook の整備により、以下の効果を期待する。

- 要求・仕様・設計に関する認識齟齬の削減
- 開発初期からの手戻り抑制
- AI Agent に対する入力品質向上による成果物品質の安定化
- ドキュメント構成・記述ルールの共通化による知見の再利用促進
- 属人化・ブラックボックス化の抑制
- 新規参画者の早期立ち上がり
- ライフサイクル全体での生産性向上

可能な範囲で、以下のような指標により効果を測定する。

| 指標                                 | 現状   | 目標    | 測定方法                       |
| ------------------------------------ | ------ | ------- | ------------------------------ |
| 要件定義〜基本設計における手戻り件数 | 未測定 | 30%削減 | 変更管理票・課題管理票の比較   |
| AIによる実装タスクの再実行率         | 未測定 | 20%削減 | 再指示・再生成回数の記録       |
| AI生成成果物のレビュー指摘件数       | 未測定 | 25%削減 | レビュー指摘件数の集計         |
| 新規参画者の立ち上がり期間           | 未測定 | 50%短縮 | 初回成果物提出までの日数を計測 |
| 主要ドキュメントの整備率             | 未測定 | 90%以上 | 必須文書の整備状況を確認       |
| 変更時の関連文書更新率               | 未測定 | 90%以上 | 変更時の文書更新状況を確認     |

## 5. 前提条件

SpecDojo Handbook の策定および活用にあたっては、以下を前提とする。

- SpecDojo は、人間の判断や責任を代替するものではなく、人とAIが協働するための基盤であること
- 対象範囲は、企画、要件定義、設計、実装、テスト、運用・保守を含むシステムライフサイクル全体とすること
- ドキュメントは、人間にとっての理解容易性と、AIにとっての解釈容易性を両立すること
- 記述ルールは、現場で継続運用可能な粒度・負荷に収めること
- Handbook は文書規約にとどまらず、テンプレート、サンプル、運用ルール、補助ツール、AI向け指示テンプレート等を含むこと
- 特定の開発手法、言語、基盤、生成AI製品に過度に依存しないこと
